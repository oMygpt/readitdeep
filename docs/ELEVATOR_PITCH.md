## 💰 版本一：投资人版

> **商业视角 | 适用于：路演、融资会议、商业计划书**

### 一句话

> **"我们正在打造科研领域的 Notion + Figma——AI 驱动的团队知识协作平台。"**

### 30 秒版

**市场痛点**：全球有 800 万科研人员，每人每年平均阅读 300+ 篇论文。但现有工具（Zotero、Mendeley）只解决"存储"问题，不解决"理解"和"协作"问题——90% 的阅读洞察锁在个人脑子里，团队知识无法复用。

**我们的产品**：**Read it DEEP** 是一个 AI 驱动的团队科研阅读平台。它用 AI 自动解析论文、提取核心方法和数据集、支持团队实时协作标注——把"被动阅读"变成"主动知识构建"。

**商业模式**：
- **B2C**：科研人员个人订阅（AI 配额制）
- **B2B**：实验室 / 研究院团队订阅（无限席位 + 私有部署）

**竞争壁垒**：
1. **LangGraph 多 Agent 架构**——5 个 AI Agent 并行提取，精度远超单次提问
2. **本地 LLM 优先**——支持私有部署，满足科研数据合规需求
3. **知识图谱沉淀**——用得越久，团队知识库价值越高（网络效应）

**里程碑**：
- ✅ 核心功能完成（解析 / 分析 / 翻译 / 协作）
- ✅ Docker 一键部署 + GHCR 自动构建
- 🔜 目标：100 个付费团队 / 1000 个月活用户

### 市场数据

| 指标 | 数据 |
|:--|:--|
| 全球科研人员 | ~800 万 |
| 年论文产出 | ~300 万篇 |
| Notion/Figma TAM 类比 | 科研垂直 SaaS 市场 $10B+ |
| 竞品（Elicit、Semantic Scholar） | 只做搜索/发现，不做深度阅读 + 协作 |

### 为什么是我们？

- **技术壁垒**：LangGraph 多 Agent 系统，不是简单的 GPT Wrapper
- **数据壁垒**：知识图谱越用越有价值，形成网络效应
- **合规优势**：支持本地 LLM 私有部署，满足科研机构数据安全需求

## 👤 版本二：用户版

> **场景化语言 | 适用于：产品官网、用户引导、社交媒体**

### 一句话

> **"用 AI 帮你读论文——自动翻译、智能提取、团队共享，读一篇胜过读十篇。"**

### 30 秒版

**你是否有这样的困扰？**
- 打开论文 PDF，公式图表乱糟糟，读一半就放弃？
- 读完一篇，三天后只记得标题？
- 团队开会讨论论文，每个人理解都不一样？

**Read it DEEP 帮你解决。**

### 核心功能

| 你想要的 | 我们怎么做 |
|:--|:--|
| **"这论文太长，给我总结一下"** | AI 自动生成论文概要、研究方法、数据集清单 |
| **"这公式什么意思？"** | 选中公式 → 一键解析 → 费曼式教学讲解 |
| **"我英语一般"** | 全文翻译，原文/译文一键切换，翻译后永久保存 |
| **"我想和实验室同学一起看"** | 创建团队 → 分享论文 → 一起高亮、批注、讨论 |
| **"这论文引用了谁？谁引用了它？"** | 知识图谱一目了然，点击直接跳转 |

### 使用流程

```
1. 拖入 PDF → 自动解析（30 秒）
2. 点击"概览" → 看 AI 总结、方法、数据集
3. 点击"开始阅读" → 沉浸式三栏布局（图谱 / 正文 / 工作台）
4. 选中任意文字 → 弹出智能操作（解析 / 翻译 / 笔记 / 高亮）
5. 分享到团队 → 队友可以看到你的标注和笔记
```

### 功能亮点

#### 🔬 智能解析
- 拖入 PDF，30 秒完成解析
- 公式完美渲染（KaTeX）
- 图片自动提取和重写

#### 🤖 AI 分析
- 自动提取研究方法、数据集、代码仓库
- 选中文字 → 弹出智能操作
  - 📐 **公式解析**：逐步推导 + 变量说明
  - 🎓 **费曼教学**：用简单语言讲明白
  - 🔍 **深度研究**：学术视角深入分析
  - 💬 **Chat with PDF**：对话式问答

#### 🌍 全文翻译
- 后台异步翻译，不阻塞阅读
- 原文/译文一键切换
- 翻译进度实时显示

#### 👥 团队协作
- 创建团队、邀请成员
- 论文一键分享 / 批量分享
- 多人高亮（按用户颜色区分）
- 讨论线程 + @提及队友
- AI 助手随时待命

#### 🕸️ 知识图谱
- 可视化引用关系
- 被引论文 / 参考文献一目了然
- 点击节点展开二级关系

### 一句话打动你

> **"读一篇论文，记住的不只是内容，还有方法、数据集、灵感——全都变成你的资产。"**

## 🛠 版本三：技术社区版

> **开发者视角 | 适用于：GitHub README、技术博客、开发者社区**

### 一句话

> **"基于 LangGraph 的论文分析 Agent 系统——多 Agent 并行执行 + 本地 LLM 支持 + Docker 一键部署。"**

### 30 秒版

**这是什么？**
一个开源友好的 AI 论文阅读平台，核心亮点：

1. **LangGraph 多 Agent 架构**——5 个 Agent（Summary / Method / Dataset / Code / Structure）并行分析，使用 `Annotated[List, operator.add]` 支持并发写入状态
2. **本地 LLM 优先**——兼容 vLLM / Ollama，支持 OpenAI 兼容 API
3. **高精度 PDF 解析**——集成 Mineru V4，公式/图表/代码完美保留
4. **向量语义检索**——pgvector 存储段落 Embedding，支持相似论文发现

### 技术栈

```
┌─────────────────────────────────────────────────────────────┐
│                       Frontend                               │
│  React + Vite + TanStack Query + Tailwind + Shadcn/UI       │
│  ReactFlow (知识图谱) + KaTeX (公式) + ReactMarkdown        │
├─────────────────────────────────────────────────────────────┤
│                        Backend                               │
│  Python 3.11+ + FastAPI + SQLAlchemy (Async)                │
│  LangGraph (多 Agent) + LangChain (LLM 抽象)                │
├─────────────────────────────────────────────────────────────┤
│                       Services                               │
│  Mineru API (PDF 解析) + Semantic Scholar + OpenAlex        │
│  pgvector (向量存储) + Redis (缓存)                         │
├─────────────────────────────────────────────────────────────┤
│                      Deployment                              │
│  Docker Compose + GHCR 自动构建 + Nginx 反代                │
└─────────────────────────────────────────────────────────────┘
```

#### API 设计

| 端点 | 功能 |
|:--|:--|
| `POST /papers` | 上传 PDF，触发后台解析 |
| `GET /papers/{id}/analysis` | 获取 AI 分析结果 |
| `POST /papers/{id}/translate` | 触发流式翻译 |
| `GET /papers/{id}/graph` | 获取引用关系图 |
| `POST /workbench/analyze/method` | 智能方法提炼 |
| `POST /teams` | 创建团队 |
| `POST /papers/{id}/annotations` | 创建协作标注 |
| `POST /ai/assist` | @AI 讨论助手 |

### 快速部署

```bash
# 克隆项目
git clone https://github.com/oMygpt/readitdeep.git
cd readitdeep

# 配置环境变量
cp .env.docker.example .env
# 编辑 .env，填写 LLM_BASE_URL 和 LLM_API_KEY

# 一键启动
docker compose -f docker-compose.ghcr.yml pull
docker compose -f docker-compose.ghcr.yml up -d

# 访问 http://localhost:3000
```

### 本地开发

```bash
# Backend
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8080

# Frontend
cd frontend
npm install
npm run dev
```
