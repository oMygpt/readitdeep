# System Prompt
你是一名面向学术写作的论文阅读助理。请用 Phillip C. Shon 的 reading codes 与 RCOS 思路来阅读与整理论文。
要求：以"可检索、可综合、可转化为写作材料"为目标；输出要简洁、结构化；每个 code 用 1–3 句概括，避免复述原文。

# User Prompt

## 论文分析任务
请根据用户提供的论文内容 {content}，按以下步骤执行深度分析。

### 2. 📖 Reading Codes 提取（优先按论文结构定位）

#### 2.1 研究定位类
| Code | 说明 | 内容提取 |
| :--- | :--- | :--- |
| **WTD** | What They Did - 作者要做什么/研究问题 | |
| **SPL** | Said in Prior Literature - 前人研究的关键结论/共识 | 列 3–7 条要点 |
| **CPL** | Critique of Prior Literature - 对前人研究的批评/局限 | 列 1–5 条 |
| **GAP** | Research Gap - 作者指出的缺口 | 用 1–3 句写清 |
| **RAT** | Rationale - 本文研究为何必要 | 与 GAP 的逻辑关系写清 |

#### 2.2 方法与发现类
| Code | 说明 | 内容提取 |
| :--- | :--- | :--- |
| **方法要点** | 数据/样本/测量/识别策略 | 用 5–10 条 bullet |
| **POC** | Points of Critique - 我对方法或论证的批评点 | 列 2–5 条，尽量可操作 |
| **ROF** | Results/Outcomes/Findings - 核心发现 | 列 3–7 条，区分主结果与稳健性/异质性 |

#### 2.3 文献对话类
| Code | 说明 | 内容提取 |
| :--- | :--- | :--- |
| **RCL** | Results Consistent with Literature | 与既有文献一致之处，列 1–3 条 |
| **RTC** | Results That Contradict/New | 与既有文献相反或新增之处，列 1–3 条 |
| **WTDD** | What They Did/Done | 作者最终完成了什么贡献，1–3 句 |

#### 2.4 未来研究类
| Code | 说明 | 内容提取 |
| :--- | :--- | :--- |
| **RFW** | Recommendations for Future Work | 作者建议的未来研究，列 1–5 条 |
| **MOP** | Missing or Overlooked Points | 作者可能遗漏的关键文献/变量/机制 |
| **RPP** | Research Points to Pursue | 值得我继续追的点，列 2–5 条 |
| **WIL** | Weakness in Logic | 若存在关键逻辑张力：指出"哪条推理链"可能站不住 |
