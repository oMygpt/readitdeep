# 参与 Read it DEEP 贡献

首先，感谢您抽出时间参与贡献！

以下是为 Read it DEEP 贡献的一套指南。这些主要是指导方针，而非硬性规定。请运用您的最佳判断，并随时在 PR 中提出对本文档的修改建议。

## 行为准则

本项目及所有参与者都受 [Read it DEEP 行为准则](CODE_OF_CONDUCT_CN.md) 的约束。参与本项目即表示您同意遵守该准则。

## 我该如何贡献？

### 报告 Bug

本节将指导您如何提交 Bug 报告。
- **使用清晰且描述性的标题**来标识问题。
- **描述重现问题的具体步骤**，越详细越好。
- **提供具体示例**来演示这些步骤。

### 建议改进

本节将指导您如何提交改进建议，包括全新的功能和对现有功能的微调。
- **使用清晰且描述性的标题**来标识建议。
- **分步描述建议的改进内容**，越详细越好。
- **解释为什么这个改进对大多数用户有用**。

### 提交 Pull Request (PR)

1. fork 本仓库并从 `main` 分支创建您的分支。
2. 如果您添加了需要测试的代码，请添加测试用例。
3. 确保测试套件通过。
4. 确保您的代码符合 lint 规范。
5. 为任何新的源代码文件添加许可证头 (AGPL v3)。
6. 提交您的 PR！
